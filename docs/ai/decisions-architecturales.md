# Décisions architecturales & patterns

- **React 19 & Compiler :** 
  - Utilisation de `babel-plugin-react-compiler` pour l'optimisation automatique.
  - **INTERDICTION** d'utiliser `useMemo` et `useCallback` manuellement (sauf cas exceptionnel non géré par le compilateur).
  - **Exception documentée** : **`ListeActivites.tsx`** — au minimum **`useMemo`** pour **`lieuxFiltrésParEmplacement`** et **`useEffect`** pour réinitialiser **`formLieuId`** si le lieu choisi ne correspond plus au filtre emplacement ; **fenêtre calendrier** (jours visibles, navigation période, clamp sur les bornes du séjour) via le hook **`useCalendrierFenetreJours`** (`listeActivitesUtils` : **`bornesDebutFenetreCalendrier`**, etc.) ; le fichier contient en outre d’autres **`useMemo` / `useCallback` / effets** (agrégations calendrier, filtres liste, etc.) pour la perf et des props stables vers **`ListeActivitesListe`** / **`ListeActivitesCalendrier`**. Ne pas généraliser ce pattern au reste de l’app sans besoin mesuré.
  - **Exception documentée** : `DetailsSejourActivites.tsx` — **`useMemo`** pour la liste **`membresEquipePourActivites`** (directeur + équipe, triée) passée à **`ListeActivites`**.
  - **Exception documentée** : `DetailsSejourOverview.tsx` — **`useCallback`** pour **`handleAccordionReorder`** (fin de glisser-déposer des panneaux) ; **`useLayoutEffect`** pour resynchroniser **`accordionOrder`** depuis **`localStorage`** au changement de **`sejourId`** (évite flash ordre par défaut). **Accordéons** : état initial **`openAccordions = []`** (tous fermés) ; ouverture via **`location.state.openAccordion`** uniquement.
  - **Exception documentée** : **`Header.tsx`** — **`useLayoutEffect`** + **`observeSiteHeaderHeight`** (`helpers/siteHeaderHeight.ts`) : **`ResizeObserver`** sur **`<header>`** → CSS **`--site-header-height`** (padding **`body`**, barres fixées type **`ListeActivites`** qui mesurent déjà le header). Lien **Profil** : rôle affiché via **`libelleRoleBadgeProfil`** (voir helper **`libelleRoleSurSejour.ts`**).
  - **Exception documentée** : **`DetailsSejourParametrage.tsx`** — **`useCallback`** pour **`handleAccordionReorder`** et **`refreshLoaderData`** (réordonnancement accordéons + **`useRevalidator`** après mutations hors **`ListeTypesActivite`** qui révalide déjà en interne).
  - **Exception documentée** : **`ListePlanningsOrganisation.tsx`** — **`useMemo`** pour données dérivées (lignes triées, blocs de sections, grilles triées alphabétiquement, liste **`membresListeModaleCelluleMembreEquipe`** lorsque l’animateur édite des cellules **`MEMBRE_EQUIPE`**, liste **`joursDuSejourPlanning`** pour la fenêtre calendrier, etc.) ; hook **`useCalendrierFenetreJours`** pour aligner la vue tableau sur **1 / 3 / 7 jours** + navigation (comme **`ListeActivites`**) ; gros écran, interactions DnD et table.
  - **Exception documentée** : **`SectionReunionsSejour.tsx`** — **`useMemo`** pour **`peutGererEcritures`** (dérivé Redux + props), **`texteCherchableParId`** (map id → texte ODJ + contenu TipTap pour filtres) et **`reunionsAffichees`** ; **`useCallback`** pour **`reinitialiserFiltres`** et **`recharger`** (liste comptes rendus). Prop **`initialReunions?`** : données du loader parent (**`detailsSejourLoader`**) — état initial trié, pas de fetch au montage ni skeleton ; **`recharger`** après CRUD.
  - **Exception documentée** : **`ListeChambres.tsx`** — **`useMemo`** / **`useCallback`** / **`useEffect`** (filtres groupes, index affectations, listes éligibles picker, sync état local depuis loader) ; mutations CRUD / référents / occupants → **`rafraichirChambres`** (1 GET **`getChambresDuSejour`**, état local — pas **`useRevalidator`**) pour éviter rechargement complet du loader parent.
  - **Exception documentée** : **`CheckboxFilterDropdown.tsx`** — **`useRef`** + **`useEffect`** (listener `mousedown` sur `document` pour fermer le panneau au clic extérieur, cleanup au démontage) ; **`useMemo`** pour le résumé (`libelleResumeMultiselectFilter`). Gestion d’un état d’ouverture local + a11y `role="listbox"`.
  - Pas d'import React inutile (React 19 auto-import).
- **Routing (singleton & connexion) :**
  - **`createBrowserRouter`** : instancier **une seule fois** au niveau **module** dans **`App.tsx`** (pas dans le corps du composant — recréation = rechargement des loaders).
  - **Connexion (`/`)** : **`loginAction`** → **`redirect(chargerProfilEtCheminDestinationApresConnexion())`** (profil + URL mémorisée **`enjoy.cheminApresConnexion`** ou accueil **`cheminAccueilDepuisEtatActuel`**) ; loader **`/`** (GET déjà authentifié) → **`chargerProfilEtCheminAccueil()`** sans consommer le chemin mémorisé ; **`connexionShouldRevalidate`** : **`false`** si **`formMethod === "POST"`**. **Interdit** **`<Navigate>`** client sur **`Connexion`**. **401** : redirect **`/`** — loaders (**`logoutAndRedirectToConnexion`**, **`RedirectToConnexion`**) + **intercepteur Axios global** (**`caller.service.ts`**) : après échec refresh ou 401 sans retry → logout, mémorisation URL, **`window.location.replace('/')`** (sauf échec login sur **`/`**) ; pas de page « Connexion requise » ni message Spring brut (« Full authentication… ») affiché sans redirect.
- **Data Fetching :** 
  - Privilégier les loaders React Router (`useLoaderData` / **`useRouteLoaderData`** pour une route ancêtre) plutôt qu’un `useEffect` de chargement initial.
  - Sous-routes du détail séjour : loader parent **`id: "sejour-detail"`** ; pages enfants lisent **`useRouteLoaderData("sejour-detail")`** sauf **Menus** / **Sanitaire** / **Chambres** (loaders dédiés + **`useLoaderData`** local). **`detailsSejourLoader`** inclut aussi **`reunions`** (**`sejourReunionService.listerReunions`**) pour la vue générale — passées à **`SectionReunionsSejour`** via **`initialReunions`** (évite double fetch / skeleton).
  - **Loaders enfants & cache séjour (RR 6.17)** : **`sejourDetailRouteCache.ts`** — **`mettreEnCacheSejourRoute(params.id, sejour)`** à la fin de **`detailsSejourLoader`** ; loaders enfants (**`menusLoader`**, futurs) lisent **`lireSejourDepuisCacheRoute`** pour dates / id sans refetch séjour. **Menus** : **`chargerDonneesMenusSejour`** = 2 appels parallèles ; **ne pas** recharger refs + menus dans un **`useEffect`** au montage du composant (Strict Mode = doublon) — privilégier le loader de route.
  - Pattern standard : exporter `async function [pageName]Loader({params}: LoaderFunctionArgs)` depuis chaque page ou module dédié (ex. **`detailsSejourLoader.ts`**, **`detailsSejourMenusLoader.ts`**).
  - Les loaders doivent propager les erreurs bloquantes via **`throwRouteLoaderError`** (`helpers/routeError.ts`) — pas de retour silencieux `[]` / `null` / `Error` inline pour les échecs réseau, auth ou serveur.
  - Sous-routes **Menus**, **Sanitaire** et **Chambres** : loaders dédiés **`menusLoader`**, **`sanitaireLoader`**, **`chambresLoader`** (`detailsSejourMenusLoader.ts`, `detailsSejourSanitaireLoader.ts`, **`detailsSejourChambresLoader.ts`**, utils **`menusSejourUtils.ts`**, cache **`sejourDetailRouteCache.ts`**).
- **Formulaires :** 
  - Utilisation stricte de `src/components/Forms/Form.tsx` (générique) piloté par une configuration (props `fields`).
  - Validation côté client via `validation` dans `FormField` (fonction qui retourne `string | null`).
  - Validation automatique des champs requis et des règles personnalisées.
  - Validation croisée : si `dateDebut` change, revalide automatiquement `dateFin` (pattern dans `handleFieldChange`).
  - Utilisation de `useRevalidator` de React Router pour rafraîchir les données après soumission.
  - Messages de succès/erreur configurables via props (`successMessage` peut être une fonction, `errorMessage`).
  - **`errorMessagePlacement`** (`"top"` par défaut \| `"bottom"`) : position du paragraphe d’erreur (au-dessus du formulaire ou sous la rangée de boutons, classe **`errorMessageBelowButtons`** + **`role="alert"`** si bas).
  - Support des champs personnalisés via `customComponent` dans `FormField`.
  - Modal de confirmation automatique après succès si `successMessage` est fourni (fermeture du formulaire après confirmation).
  - Gestion de l'état de chargement (`loading`, `isSubmitting`) pour désactiver les champs et boutons pendant la soumission.

Styles : importer **`PlanningCalendrier.module.scss`** (classes exportées selon besoin du consommateur). **Écrans** : **`ListeActivites*`**, **`DetailsSejourMenus`** (formulaire **`MenuRepasFormulaireCorps`**, helpers **`menuRepas.ts`**).

Option **`runningHeaderLayout: 'title-page-split'`** (opt-in, ex. **`DossierEnfant`**) : titre seul en **`@top-left`**, **n/N** en **`@top-right`**, marge haute renforcée — **sans** impact sur les autres consommateurs (layout **`default`**). **Firefox** : **`supportsPageMarginBoxes()`** (`detectPrintCapabilities.ts`) → **`runningHeaderMode: 'fixed'`** + **`PrintRunningHeader`** (titre répété, **sans numéros**, même typo ; flex split si **`title-page-split`**).

L’option « En-têtes et pieds de page » du dialogue navigateur reste hors contrôle CSS (à décocher côté utilisateur si besoin).

## Helpers & utilitaires

- **Gestion des dates :**
  - `formaterDate(date)` : Formate une date pour l'affichage (toLocaleDateString). Gère strings ISO, timestamps (secondes/millisecondes), Date objects. Retourne "Date invalide" si invalide, "" si null/undefined.
  - `parseDate(date)` : Parse une date en Date object, gère tous les formats (ISO, timestamp secondes/millisecondes, Date, YYYY-MM-DD). Retourne null si invalide.
  - `dateToISO(date)` : Convertit une date en string ISO 8601 pour l'API. Gère tous les formats (ISO, timestamp, Date, YYYY-MM-DD). Retourne undefined si invalide/absente.
  - `formatDateAnglais(date)` : Formate une date au format YYYY-MM-DD (format anglais pour les inputs date). Gère timestamps (secondes/millisecondes), Date objects, strings ISO.
  - `calculerDureeEnJours(dateDebut, dateFin)` : Calcule la durée en jours entre deux dates (arrondi au supérieur avec Math.ceil).
