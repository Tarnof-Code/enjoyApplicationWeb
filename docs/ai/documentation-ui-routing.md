# Documentation UI — routes & navigation

## Navigation & routes

- Routes protégées via `ProtectedRoute.tsx` et Redux pour l'état d'authentification.
- Routes définies dans **`App.tsx`** : instance unique **`createBrowserRouter`** au **niveau module** (pas dans le corps du composant **`App`**) → **`RouterProvider`** seul dans **`App`**.
- Routes principales :
  - `/` : Page de connexion ; **loader** (GET / visite directe) si déjà authentifié : **`chargerProfilEtCheminAccueil()`** (**`getUser`**, puis **`redirect`** HTTP vers **`/mes-sejours/{dernierId}`** si **DIRECTION** / **BASIC_USER** et dernier séjour mémorisé ; sinon **`/profil`**) — **échec `getUser`** → page d’erreur. **`loginAction`** (POST) : après **`saveAccessToken`**, **`redirect(chargerProfilEtCheminDestinationApresConnexion())`** — URL mémorisée avant redirection vers `/` (voir **`cheminApresConnexion.ts`**) si présente, sinon même logique d’accueil que le loader ; message réseau via **`NETWORK_ERROR_MESSAGE`**. **`connexionShouldRevalidate`** (`Connexion.tsx`) : **`formMethod === "POST"`** → pas de revalidation du loader `/` (l’action gère déjà redirect + profil — évite une 2ᵉ navigation vers le séjour). **Pas de `<Navigate>` client** sur **`Connexion`** (redondant avec loader + action).
  - `/profil` : Page de profil utilisateur (accessible à tous les utilisateurs connectés) — **`Profil.tsx`** + **`Profil.module.scss`** : layout **2 colonnes** (carte **circulaire** sticky ~**23.5rem** : grande **photo** + **badge rôle** via **`libelleRoleBadgeProfil`** — rôle sur le **séjour en contexte** (**`headerSejourContext`**) si l’utilisateur y est directeur ou membre d’équipe, sinon **rôle système** ; pas de nom/email dans la carte) ; **3 accordéons** à droite (*Informations personnelles*, *Contact*, *Compte & statut*) — **1er ouvert** par défaut ; champs en lignes compactes (label + value côte à côte), édition **champ par champ** ; mot de passe via **`ChangePasswordForm`**. **Email** : modifiable **ADMIN** uniquement (auto-modification) ; **DIRECTION** / **BASIC_USER** : lecture seule + **`getEmailReadOnlyMessage`** ; **`role`** / **`dateExpirationCompte`** non éditables ici
  - `/utilisateurs` : Liste des utilisateurs (ADMIN uniquement) — édition via **`UserForm`** sans **`sejourId`** : **email**, **`role`**, **`dateExpirationCompte`** éditables
  - `/sejours` : Liste de **tous** les séjours (ADMIN uniquement)
  - `/mes-sejours` : Liste des séjours de l'utilisateur connecté — **DIRECTION** (séjours dont il est directeur) ou **BASIC_USER** (séjours dont il est membre d'équipe). Loader `mesSejoursLoader` → `sejourService.getAllSejoursByUtilisateur()` → `GET /sejours/utilisateur/{utilisateurTokenId}` (l'API filtre selon le rôle, l'ADMIN reçoit tous les séjours mais cette route ne lui est pas servie)
  - `/mes-sejours/:id` : Détail séjour — **index** = vue générale (`DetailsSejourOverview`) : bandeau **informations générales** (description, lieu, dates, durée) puis accordéons **Réunions / Équipe / Enfants / Groupes** (**fermés** par défaut). Loader **`detailsSejourLoader`** ( **`Promise.all`** : séjour, enfants, groupes, **chambres**, lieux, moments, horaires, activités, types activité, grilles planning, réunions, **`activitesPrestataires`**), id de route **`sejour-detail`**. **Accordéon équipe** (`Equipe` / **`TableauUtilisateurs`**) : colonnes **Groupe(s)** (référent, pastilles **`AffichageGroupesListe`**) et **Chambre** (type **EQUIPE**) si données ; **Rôle** abrégé ; colonnes masquables + bouton **Imprimer** (liste filtrée, **`canPrint`**, module **`src/print/`**). **Accordéon enfants** (`ListeEnfants`) : colonnes **Groupe(s)** (pastilles, couleur par groupe) et **Chambre** (type **ENFANT**, identifiant seul) si données — affectation inline (modales crayon) ; genre affiché **Garçon** / **Fille** ; cases en-tête pour masquer les colonnes (sauf Nom/Prénom) ; **Imprimer** idem. Une seule exécution attendue à l’atterrissage post-login (ne pas empiler redirect loader `/` + redirect action + **`Navigate`** client).
  - `/mes-sejours/:id/organisation` : Liste des **plannings organisation** (`DetailsSejourOrganisationLayout` → `DetailsSejourOrganisation`, route **`index`** sous **`organisation`**)
  - `/mes-sejours/:id/organisation/:grilleId` : **Éditeur** de grille pleine page (même composant **`DetailsSejourOrganisation`**, paramètre **`grilleId`**) — navigation **1 / 3 / 7 jours** ; bouton **Imprimer** (module **`src/print/`**, paysage A4 ; métadonnées : consigne globale si renseignée)
  - `/mes-sejours/:id/activites` : Même loader parent ; **Activités** (`DetailsSejourActivites` / **`ListeActivites`**) — props **`activites`** + **`activitesPrestataires`** ; bascule **Liste / Calendrier** ; calendrier : cartes sortie, conflits, **`nonParticipations`** ; **Imprimer** (vue calendrier, modale **Noir et blanc** / **Couleurs** ; métadonnées : filtres animateurs/groupes si actifs) ; lien **Sorties / prestas** vers la sous-route ci‑dessous.
  - `/mes-sejours/:id/activites/sorties-prestas` : Même loader **`sejour-detail`** ; **`DetailsSejourActivitesPrestataires`** / **`ListeActivitesPrestataires`** (CRUD sorties, filtres, exclusions animateurs regroupées nom + moments, **Historique** par carte via **`getHistoriqueActivitePrestataire`** — lecture tout participant ; écriture **`peutGererEcritures`** = **`peutGererMembresEquipeSejour`**). Retour **← Activités**.
  - `/mes-sejours/:id/menus` : Loader **`menusLoader`** (`detailsSejourMenusLoader.ts`) + **`DetailsSejourMenus`**. **2 GET** à la navigation : **`references-alimentaires-agregees-enfants`** + **`menus`** (plage = dates du séjour via **`paramsListerMenusPourSejour`** dans **`menusSejourUtils.ts`**). Le **`SejourDTO`** vient du cache **`sejourDetailRouteCache`** (rempli par **`detailsSejourLoader`**) — **pas** de 2ᵉ **`getSejourById`**. Données initiales via **`useLoaderData`** (pas de **`useEffect`** de chargement au montage → évite doublon React Strict Mode). Après création / modification / suppression d’un menu : **`rafraichirMenus`** → 1 GET **`menus`** uniquement. Bascule **Calendrier / Liste** ; navigation **1 / 3 / 7 jours** ; **Imprimer** (modale **Noir et blanc** / **Couleurs** — fonds repas = **`couleurFondCarteMenuPourTypeRepas`**, preset **`menusGrid`**).
  - `/mes-sejours/:id/sanitaire` : Loader **`sanitaireLoader`** (`detailsSejourSanitaireLoader.ts`) + **`DetailsSejourSanitaire`**. Bascule **Cahier d'infirmerie** / **Dossiers sanitaires** (vue **`enjoy.sanitaire.vue.{sejourId}`**). **Cahier d'infirmerie** : **`ListeCahierInfirmerie`** — filtres jour + recherche (insensible accents, **`normaliserPourRechercheTexte`**) ; colonnes **Date** / **heure** (2 lignes), **Enfant**, **Description**, **Localisation**, **Soins**, **Appels**, **Soigné(e) par**, **Auteur** (écran) ; CRUD entrées ; **`CahierInfirmerieForm`** — champ **Soigné(e) par** ; **Imprimer** (contenu filtré, titre **Cahier d'infirmerie — {nom} ({dates})**, sans **Auteur**, colonne **Signature de la direction**). **Dossiers** : barre **`.optionsBar`** — filtres **Allergies**, **Traitements**, **Contacts parents**, **Infos médicales & PAI**, **Alimentation**, **Autres infos**, **À prendre en sortie** (+ case **Tout**) ; prefs **`enjoy.sanitaire.opts.{sejourId}`** ; **`ListeSanitaireDossiers`** filtre lignes (**OU** entre groupes cochés) et affiche colonnes dynamiques ; filtres tableau **`sessionStorage`** ; **Imprimer** (via **`Liste`**, colonnes cochées + filtres actifs ; métadonnées : effectif séjour) ; retour dossier enfant → **`state.from === 'sanitaire'`**
  - `/mes-sejours/:id/chambres` : Loader **`chambresLoader`** + **`DetailsSejourChambres`** / **`ListeChambres`**. **1 GET** chambres ; **`enfants`**, **`groupes`**, équipe via **`sejour-detail`**. **ACCES_SEJOUR** : CRUD + référents + occupants pour tout participant ; POST affectation → merge **`ChambreDto`** retourné en état local. **Historique** par chambre : **`GET …/chambres/{chambreId}/historique`** (modal au clic **Historique**, re-fetch si ouverte après mutation).
  - `/mes-sejours/:id/parametrage` : Même loader parent ; **Paramétrage** (`DetailsSejourParametrage`) — réservé au **directeur du séjour** ou à un **adjoint** (`peutGererMembresEquipeSejour` : segment **Header** absent et **`Navigate`** vers la vue générale sinon). Lieux, Moments, Horaires, Types d’activité, **Références alimentaires** (accordéon **5** — **`ListeReferencesAlimentaires`**, catalogue global sans champ **`ordre`** visible), **Affichage des menus** (préférences locales navigateur pour l’onglet Menus).
  - `/mes-sejours/:id/enfants/:enfantId/dossier` : Dossier d’un enfant — même arbre **`sejour-detail`** ; loader **`dossierEnfantLoader`**, composant **`DossierEnfant`** (DIRECTION ou BASIC_USER participant au séjour). Affichage **4 sections** (contacts, médical, traitements, autres) ; édition par section (directeur, adjoint, **AS**). **Imprimer** : fiche complète via **`src/print/`** (titre **`Dossier sanitaire — {enfant}`**, métadonnée séjour) ; retour depuis sanitaire → **`state.from === 'sanitaire'`**
- **`/erreur`** : Page d’erreur programmatique (**`Erreur.tsx`**) — lit **`location.state`** (`AppRouteErrorPayload` depuis **`navigateToRouteError`** ou **`Layout`** si échec **`getUser`** hors 401).
- **`*` (catch-all)** : URL inconnue → loader qui lève une **404** structurée.
- **Gestion des erreurs (routes)** :
  - **`errorElement`** sur la route racine **`/`** → **`error-page.tsx`** : **401** → **`RedirectToConnexion`** (logout + **`Navigate`** vers **`/`**) ; autres erreurs → **`ErreurAffichage`** (`useRouteError` + **`classifyRouteError`**).
  - Helper central **`helpers/routeError.ts`** : **`classifyApiError`**, **`throwRouteLoaderError`** (loaders), **`navigateToRouteError`** (fetch client bloquant), **`logoutAndRedirectToConnexion`** / **`FORBIDDEN_ROUTE_ERROR`**, messages FR fixes pour **401** / **403** (ignore textes Spring type « Full authentication… » / « Access Denied »). **401** : enregistrement de l’URL courante (**`cheminApresConnexion.ts`**) puis redirect **`/`** — plus de page intermédiaire « Connexion requise ». **Tous les appels API** (y compris **`useEffect`**, mutations composants) : intercepteur **`caller.service.ts`** applique le même redirect après échec refresh (message Spring **401** non affiché à la place du redirect).
  - **`Pages/Erreurs/RedirectToConnexion.tsx`** : utilisé par boundary et route **`/erreur`** si state **401**.
  - **`ErreurAffichage`** : pas de code HTTP affiché ; bouton **Accueil** (403, 404), **Réessayer** (réseau / serveur). **`body.no-padding-top`** sur les pages d’erreur.
  - Loaders principaux (**`mesSejoursLoader`**, **`profilLoader`**, listes admin, **`detailsSejourLoader`**, **`dossierEnfantLoader`**, **`menusLoader`**, **`sanitaireLoader`**, **`chambresLoader`**) : **`throwRouteLoaderError`** au lieu de renvoyer `[]` / `null` / `Error` inline.
  - **`ProtectedRoute`** : rôle manquant + connecté → **Chargement…** ; mauvais rôle → **`ErreurAffichage`** (**403**) ; non connecté → mémorisation URL (**`enregistrerCheminApresConnexionDepuisLocation`**) puis **`/`**.
  - **Retour après connexion** : **`sessionStorage`** clé **`enjoy.cheminApresConnexion`** ; chemins internes valides uniquement (pas **`/`**, pas **`/erreur`**, pas d’URL absolue) ; consommé par **`chargerProfilEtCheminDestinationApresConnexion`** après **`loginAction`** ; effacé à la déconnexion volontaire (**`Header`** → **`effacerCheminApresConnexion`**).
  - Réseau : **`NETWORK_ERROR_MESSAGE`** / **`isNetworkError`** dans **`axiosError.ts`** ; **`adaptAxiosError`** lève un **`Error`** FR si pas de **`response`** (plus de « Network Error » brut).
- Permissions des routes `/mes-sejours/*` : **`[RoleSysteme.DIRECTION, RoleSysteme.BASIC_USER]`** (l'API filtre les séjours selon le rôle ; un BASIC_USER ne voit que les séjours où il est membre d'équipe).
- Utilisation de `useNavigate` pour la navigation programmatique.
- Actions React Router pour les formulaires (ex. **`loginAction`** dans **`Connexion.tsx`**) ; **`shouldRevalidate`** sur la route **`/`** (**`connexionShouldRevalidate`**) pour éviter double chargement après POST.
- Layout (`Layout.tsx`) :
  - Header affiché uniquement si `pathname !== "/"` et `role !== null`
  - Pour les rôles **DIRECTION** et **BASIC_USER** sur `/mes-sejours/:id*`, le **Header** utilise **`useMatch`** + **`useRouteLoaderData("sejour-detail")`** (données valides, pas `Error`) pour le **fil** (nom du séjour en pill), les segments **Vue générale** / **Organisation** / **Activités** / **Menus** / **Sanitaire** / **Chambres** et (**si directeur ou adjoint du séjour**) **Paramétrage**, ainsi que le lien **Mes séjours** (`Header.tsx`, drapeau interne **`isParticipantSejour`**)
  - **Header responsive** : **`Navbar expand="xxl"`** — menu burger **&lt; 1400px** (`Header.module.scss`, **`$header-nav-burger-max: 1399.98px`**) ; hauteur du bandeau mesurée en JS (**`observeSiteHeaderHeight`**) → **`--site-header-height`** pour **`body { padding-top }`** et composants à barre fixe (ex. **`ListeActivites`**, variable locale **`--liste-act-site-header`**). Lien **Profil** : **`Prénom (rôle)`** via **`libelleRoleBadgeProfil`** + snapshot séjour **`resolvedForHeader`** (même règle que badge Profil)
  - Footer actuellement commenté
  - Gestion automatique du padding-top du body : par défaut **`var(--site-header-height)`** (`styles.scss`) ; **`/`**, rôle null, **`/erreur`** → classe **`no-padding-top`** (padding **0**)
  - Récupération automatique du profil utilisateur si connecté mais rôle manquant dans Redux ; échec **401** → mémorisation URL + logout + **`navigate("/")`** ; autre erreur → **`/erreur`**
